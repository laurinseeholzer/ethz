\section{Hardware Support for Parallelism}

Provides $O(1)$ space locks via \textbf{atomic instructions} (Java: \texttt{AtomicInteger}, etc). 

\begin{definition}
    \begin{itemize}
        \item \textbf{Compare and Swap (CAS):} Atomically update memory if equal to expected value.
        \item \textbf{Test and Set (TAS):} Atomically read memory and set to 1.
        \item \textbf{LL/SC:} LL sets "link". SC writes only if link still valid (no other writes).
    \end{itemize}
\end{definition}

\subsection{Atomic Locks}

\textbf{TAS Lock:} \texttt{while(state.getAndSet(true));} \\
Causes high cache coherence traffic (continuous atomic updates).

\textbf{TATAS Lock:} Spin on \textit{read} before trying \textit{write}.
\begin{codeexample}
public void lock() {
    do { while (state.get()); } 
    while (state.getAndSet(true));
}
\end{codeexample}

\begin{remark}
    \textbf{Backoff Strategy:} E.g., exponential wait after failed acquire, significantly reduces cache traffic and contention.
\end{remark}
